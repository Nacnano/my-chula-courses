Pick a choice:
การสื่อสารแบบ Asynchronous ฝั่ง client ไม่ต้องรอ response แต่สามารถทำงานอื่นได้ทันทีหลังส่ง request ให้ server
การสื่อสารแบบ Synchronous สามารถสื่อสารได้แบบ one-to-one เท่านั้น
ในการสื่อสารแบบ publish/subscribe และ การสื่อสารแบบ publish/async response เหมือนกันที่ฝั่ง client ส่ง message ไปยังหลาย server ในคราวเดียว
การสื่อสารแบบ request/response สามารถเป็นได้ทั้งแบบ synchronous และ asynchronous
One-way Asynchronous หมายถึง server ส่ง response กลับมาโดยไม่ต้องมี request จาก client

อาจใช้วิธี Network timeouts คือการจำกัดเวลาที่รอ response จาก remote services หากไม่ได้รับ response ตามเวลาที่กำหนดจะยุติการรอ
